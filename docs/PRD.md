## 👥 사용자 스토리

### 사용자 역할
- **관리자 (Admin)**: 전체 프로젝트 관리, 사용자 권한 설정
- **팀원 (Member)**: 태스크 생성/수정, 댓글 작성
- **게스트 (Guest)**: 읽기 전용 권한

### 주요 기능별 사용자 스토리

#### 1. 인증 & 권한
- [ ] 사용자는 이메일/비밀번호로 회원가입할 수 있다
- [ ] 사용자는 로그인/로그아웃할 수 있다
- [ ] 사용자는 OAuth(Google/GitHub)로 로그인할 수 있다
- [ ] 사용자는 비밀번호를 재설정할 수 있다
- [ ] 관리자는 사용자 역할을 변경할 수 있다

#### 2. 태스크 관리
- [ ] 사용자는 태스크를 생성할 수 있다
- [ ] 사용자는 태스크를 수정/삭제할 수 있다
- [ ] 사용자는 태스크에 담당자를 지정할 수 있다
- [ ] 사용자는 태스크 상태를 변경할 수 있다 (TODO/IN_PROGRESS/DONE)
- [ ] 사용자는 태스크를 필터링/검색할 수 있다

#### 3. 실시간 기능
- [ ] 사용자는 실시간으로 태스크 업데이트를 받는다
- [ ] 사용자는 실시간 채팅을 할 수 있다
- [ ] 사용자는 다른 사용자의 온라인 상태를 볼 수 있다

## 🚫 비기능 요구사항

### 성능
- **응답 시간**: API 응답 < 200ms (평균)
- **페이지 로드**: 초기 로드 < 3초
- **동시 접속**: 최소 100명 지원

### 보안
- HTTPS 필수
- JWT 토큰 기반 인증
- XSS, CSRF 방어
- Rate Limiting (100 req/min per user)
- 민감한 데이터 암호화

### 가용성
- Uptime 99.9% 목표
- 자동 백업 (일 1회)
- 에러 모니터링 및 알림

## 📊 데이터 모델 개요

### 주요 엔티티
1. **User** (사용자)
2. **Project** (프로젝트)
3. **Task** (태스크)
4. **Comment** (댓글)
5. **Notification** (알림)

### 관계
```
User 1:N Project (사용자는 여러 프로젝트를 가질 수 있음)
Project 1:N Task (프로젝트는 여러 태스크를 가질 수 있음)
Task 1:N Comment (태스크는 여러 댓글을 가질 수 있음)
User 1:N Notification (사용자는 여러 알림을 받을 수 있음)
```
