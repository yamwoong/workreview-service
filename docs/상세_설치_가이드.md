## 🔧 사전 준비사항

### 1. Node.js 설치
**Windows:**
1. [Node.js 공식 웹사이트](https://nodejs.org/) 방문
2. LTS (Long Term Support) 버전 다운로드
3. 설치 파일 실행 및 기본 설정으로 설치
4. 설치 완료 후 터미널에서 `node --version` 확인

**macOS:**
```bash
# Homebrew를 사용한 설치
brew install node
```

**Linux (Ubuntu/Debian):**
```bash
# NodeSource repository 추가
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -

# Node.js 설치
sudo apt-get install -y nodejs
```

### 2. MongoDB 설치

#### 옵션 A: 로컬 MongoDB 설치

**Windows:**
1. [MongoDB 다운로드 센터](https://www.mongodb.com/try/download/community) 방문
2. Windows용 설치 파일 다운로드
3. 설치 시 "Install MongoDB as a Service" 옵션 선택
4. MongoDB Compass (GUI 도구) 함께 설치 권장

**macOS:**
```bash
# Homebrew를 사용한 설치
brew tap mongodb/brew
brew install mongodb-community
brew services start mongodb-community
```

**Linux (Ubuntu/Debian):**
```bash
# MongoDB 공식 패키지 추가
wget -qO - https://www.mongodb.org/static/pgp/server-5.0.asc | sudo apt-key add -
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/5.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-5.0.list

# 설치
sudo apt-get update
sudo apt-get install -y mongodb-org

# 시작
sudo systemctl start mongod
sudo systemctl enable mongod
```

#### 옵션 B: MongoDB Atlas (클라우드)

1. [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) 접속
2. 무료 계정 생성 (Free Tier 사용 가능)
3. 새 클러스터 생성 (M0 Sandbox - 무료)
4. Database Access에서 사용자 생성
5. Network Access에서 IP 주소 허용 (0.0.0.0/0 - 개발용)
6. 연결 문자열 복사 (나중에 `.env`에 사용)

### 3. Git 설치
**Windows:**
- [Git for Windows](https://git-scm.com/download/win) 다운로드 및 설치

**macOS:**
```bash
brew install git
```

**Linux:**
```bash
sudo apt-get install git
```

### 4. Gmail 앱 비밀번호 생성
이메일 기능(비밀번호 재설정 등)을 위해 필요합니다.

1. [Google 계정 설정](https://myaccount.google.com/) 접속
2. 보안 → 2단계 인증 활성화 (미활성화 상태인 경우)
3. 보안 → 앱 비밀번호 생성
4. "메일" 및 "Windows 컴퓨터" 선택 (또는 "기타")
5. 생성된 16자리 비밀번호 복사 (나중에 `.env`에 사용)

#### backend/.env 파일 내용 작성
에디터로 `.env` 파일을 열고 다음 내용을 작성하세요:

```env
# ==========================================
# 서버 설정
# ==========================================
PORT=5000
NODE_ENV=development

# ==========================================
# 데이터베이스 설정
# ==========================================
# 로컬 MongoDB 사용 시:
DATABASE_URL=mongodb://localhost:27017/workreview

# MongoDB Atlas 사용 시:
# DATABASE_URL=mongodb+srv://username:password@cluster.mongodb.net/workreview?retryWrites=true&w=majority

# ==========================================
# JWT 토큰 설정
# ==========================================
# 주의: 프로덕션에서는 반드시 강력한 시크릿 키로 변경하세요!
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production-min-32-chars
JWT_EXPIRES_IN=15m

JWT_REFRESH_SECRET=your-super-secret-refresh-key-change-this-too-min-32-chars
JWT_REFRESH_EXPIRES_IN=7d

# ==========================================
# 프론트엔드 URL (CORS 설정)
# ==========================================
FRONTEND_URL=http://localhost:5173

# ==========================================
# 이메일 설정 (Gmail SMTP)
# ==========================================
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your-email@gmail.com
EMAIL_PASSWORD=your-16-digit-app-password
EMAIL_FROM=WorkReview <noreply@workreview.com>

# ==========================================
# 보안 설정 (선택사항)
# ==========================================
# 비밀번호 해싱 복잡도 (기본값: 12)
BCRYPT_SALT_ROUNDS=12

# Rate Limiting (요청 제한)
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX_REQUESTS=100
```

### MongoDB Atlas 사용 시

#### 1. 연결 문자열 복사
MongoDB Atlas 대시보드에서:
1. Clusters → Connect
2. Connect your application
3. Driver: Node.js, Version: 4.1 or later
4. 연결 문자열 복사:
```
mongodb+srv://username:<password>@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
```

#### 2. .env 파일 업데이트
```env
DATABASE_URL=mongodb+srv://username:your_actual_password@cluster0.xxxxx.mongodb.net/workreview?retryWrites=true&w=majority
```

주의:
- `<password>`를 실제 비밀번호로 변경
- 데이터베이스 이름 (`workreview`) 추가
- 특수문자가 비밀번호에 있으면 URL 인코딩 필요

## ✅ 테스트 방법

### 1. 회원가입 테스트

#### 단계:
1. `http://localhost:5173/register` 접속
2. 다음 정보 입력:
   - 이름: `홍길동`
   - 이메일: `test@example.com`
   - 비밀번호: `Test1234!` (최소 8자, 대소문자, 숫자 포함)
   - 비밀번호 확인: `Test1234!`
   - 역할: `employee` (직원)
3. "회원가입" 버튼 클릭
4. 성공 시 자동으로 로그인되고 홈페이지로 이동

#### 확인사항:
- [ ] 입력 유효성 검사 작동 (빈 필드, 잘못된 이메일 형식 등)
- [ ] 중복 이메일 확인
- [ ] 회원가입 성공 메시지 표시
- [ ] 자동 로그인 및 리다이렉션

### 2. 로그인 테스트

#### 단계:
1. 로그아웃 (프로필 → 로그아웃)
2. `http://localhost:5173/login` 접속
3. 이메일: `test@example.com`
4. 비밀번호: `Test1234!`
5. "로그인" 버튼 클릭

#### 확인사항:
- [ ] 로그인 성공 메시지
- [ ] 토큰 저장 확인 (개발자 도구 → Application → Local Storage)
- [ ] 홈페이지로 리다이렉션
- [ ] 사용자 정보 표시

### 3. 프로필 수정 테스트

#### 단계:
1. 로그인 상태에서 프로필 페이지 이동
2. 이름 변경: `홍길동` → `김철수`
3. "저장" 버튼 클릭
4. 페이지 새로고침하여 변경사항 확인

#### 확인사항:
- [ ] 프로필 수정 성공 메시지
- [ ] 변경사항 즉시 반영
- [ ] 데이터베이스에 저장 확인

### 4. 비밀번호 변경 테스트

#### 단계:
1. 프로필 페이지 → 비밀번호 변경 섹션
2. 현재 비밀번호: `Test1234!`
3. 새 비밀번호: `NewPass1234!`
4. 비밀번호 확인: `NewPass1234!`
5. "변경" 버튼 클릭
6. 로그아웃 후 새 비밀번호로 로그인

#### 확인사항:
- [ ] 현재 비밀번호 검증
- [ ] 새 비밀번호 유효성 검사
- [ ] 변경 성공 메시지
- [ ] 새 비밀번호로 로그인 가능

### 5. 비밀번호 재설정 테스트 (이메일)

#### 단계:
1. 로그아웃 상태
2. `http://localhost:5173/forgot-password` 접속
3. 이메일 입력: `test@example.com`
4. "재설정 링크 보내기" 클릭
5. 이메일 확인 (Gmail 받은편지함)
6. 이메일의 재설정 링크 클릭
7. 새 비밀번호 입력 및 확인
8. "비밀번호 재설정" 클릭
9. 새 비밀번호로 로그인

#### 확인사항:
- [ ] 이메일 발송 성공 메시지
- [ ] 이메일 수신 확인
- [ ] 재설정 링크 유효성 (6자리 코드)
- [ ] 새 비밀번호로 로그인 가능
- [ ] 토큰 만료 처리 (유효기간: 1시간)

## 🐛 문제 해결

### 1. 백엔드 실행 오류

#### 문제: "Error: Cannot find module 'dotenv'"
```bash
# 해결: 의존성 재설치
cd backend
rm -rf node_modules package-lock.json
npm install
```

#### 문제: "MongooseError: The `uri` parameter to `openUri()` must be a string"
```bash
# 해결: .env 파일의 DATABASE_URL 확인
# 올바른 형식:
DATABASE_URL=mongodb://localhost:27017/workreview
```

#### 문제: "Error: connect ECONNREFUSED 127.0.0.1:27017"
```bash
# 해결: MongoDB가 실행 중인지 확인
# Windows:
net start MongoDB

# macOS:
brew services start mongodb-community

# Linux:
sudo systemctl start mongod
```

#### 문제: "JsonWebTokenError: secret or public key must be provided"
```bash
# 해결: .env 파일의 JWT_SECRET 확인
# 최소 32자 이상의 강력한 키 사용
JWT_SECRET=your-very-long-and-secure-secret-key-at-least-32-characters
```

### 2. 프론트엔드 실행 오류

#### 문제: "TypeError: Cannot read properties of undefined (reading 'VITE_API_URL')"
```bash
# 해결: .env 파일 생성 및 VITE_API_URL 설정
echo "VITE_API_URL=http://localhost:5000/api" > .env

# 개발 서버 재시작
npm run dev
```

#### 문제: "CORS policy: No 'Access-Control-Allow-Origin' header"
```bash
# 해결: 백엔드 .env의 FRONTEND_URL 확인
FRONTEND_URL=http://localhost:5173

# backend/src/config/cors.ts 확인
```

#### 문제: "Module not found: Can't resolve 'axios'"
```bash
# 해결: 의존성 재설치
cd frontend
rm -rf node_modules package-lock.json
npm install
```

### 3. 데이터베이스 오류

#### 문제: "MongoServerError: Authentication failed"
```bash
# 해결 1: MongoDB Atlas 사용 시 연결 문자열 확인
# 비밀번호에 특수문자가 있으면 URL 인코딩 필요

# 해결 2: 로컬 MongoDB 사용 시 인증 비활성화
DATABASE_URL=mongodb://localhost:27017/workreview
```

#### 문제: "MongooseError: Operation `users.insertOne()` buffering timed out"
```bash
# 해결: MongoDB Atlas IP 화이트리스트 확인
# 0.0.0.0/0 추가 (개발 환경에서만)
```

### 4. 이메일 발송 오류

#### 문제: "Error: Invalid login: 535-5.7.8 Username and Password not accepted"
```bash
# 해결: Gmail 앱 비밀번호 확인
# 1. 2단계 인증 활성화 확인
# 2. 앱 비밀번호 재생성
# 3. .env 파일 업데이트 (공백 제거)
EMAIL_PASSWORD=abcdefghijklmnop
```

#### 문제: "Error: self signed certificate in certificate chain"
```bash
# 해결: NODE_TLS_REJECT_UNAUTHORIZED 설정 (개발 환경에만)
# backend/.env에 추가
NODE_TLS_REJECT_UNAUTHORIZED=0
```

### 5. 포트 충돌 오류

#### 문제: "Error: listen EADDRINUSE: address already in use :::5000"
```bash
# 해결 1: 포트 사용 중인 프로세스 종료 (Windows)
netstat -ano | findstr :5000
taskkill /PID <PID> /F

# 해결 2: 다른 포트 사용
# backend/.env
PORT=5001

# frontend/.env
VITE_API_URL=http://localhost:5001/api
```

#### 문제: "Error: listen EADDRINUSE: address already in use :::5173"
```bash
# 해결: Vite 개발 서버 포트 변경
# frontend/vite.config.ts
server: {
  port: 5174
}
```

### 6. TypeScript 컴파일 오류

#### 문제: "TS2307: Cannot find module 'express'"
```bash
# 해결: 타입 정의 설치
cd backend
npm install --save-dev @types/node @types/express
```

### 7. 빌드 오류

#### 문제: "FATAL ERROR: Reached heap limit Allocation failed"
```bash
# 해결: Node.js 메모리 증가
# package.json scripts 수정
"build": "node --max-old-space-size=4096 ./node_modules/.bin/tsc"
```
